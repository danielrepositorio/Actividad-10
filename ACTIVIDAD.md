# Micropropuesta: Implementación de Control de Versiones con Git

## Problema
Actualmente, el equipo tiene dificultades al trabajar en conjunto porque varios miembros modifican archivos sin ningún control. Esto provoca conflictos, versiones duplicadas y, en algunos casos, pérdida de información importante.

## Evidencia
- Archivos sobrescritos por error.
- Dificultad para saber cuál es la versión más reciente.
- Tiempo perdido tratando de corregir errores o recuperar cambios.
- Problemas al trabajar simultáneamente en el mismo archivo.

## Solución
Se propone implementar el uso de Git como sistema de control de versiones.

# Guion dividido para 3 integrantes (Duración total: 3 minutos)

---

##  Integrante 1 (0:00 - 1:00) → Problema + Evidencia

Buenos días.

Actualmente, nuestro equipo enfrenta un problema importante: varias personas modifican archivos sin ningún tipo de control.

Esto provoca situaciones como:
- Archivos sobrescritos por error.
- Confusión sobre cuál es la versión correcta.
- Pérdida de información importante.
- Retrasos al intentar corregir errores.

Por ejemplo, cuando dos personas trabajan en el mismo archivo al mismo tiempo, uno puede terminar borrando el trabajo del otro sin darse cuenta.

Esto demuestra que necesitamos una mejor forma de organizar y controlar los cambios dentro del equipo.

---

##  Integrante 2 (1:00 - 2:00) → Solución

Para resolver este problema, proponemos utilizar Git como sistema de control de versiones.

Git es una herramienta que permite:
- Guardar un historial de todos los cambios realizados.
- Trabajar en equipo sin sobrescribir archivos.
- Crear diferentes versiones del proyecto mediante ramas.
- Recuperar versiones anteriores en caso de errores.

Por ejemplo, con Git, cada integrante puede trabajar en su propia versión del archivo y luego combinar los cambios de forma segura.

Esto evita conflictos y permite un trabajo mucho más organizado.

---

##  Integrante 3 (2:00 - 3:00) → Beneficios + Cierre

La implementación de Git trae varios beneficios importantes:

- Evita la pérdida de información.
- Reduce los conflictos entre integrantes.
- Mejora la organización del trabajo.
- Facilita el trabajo en equipo.
- Permite ver claramente qué cambios se han hecho.

En conclusión, Git no solo soluciona los problemas actuales del equipo, sino que también mejora la forma en la que trabajamos.

Es una herramienta muy utilizada en el desarrollo de software y nos ayudará a ser más eficientes y organizados.

# Preguntas y respuestas (Q&A)

**¿Es difícil aprender Git?**  
No, lo básico se puede aprender rápidamente y es suficiente para empezar a trabajar.

**¿Qué pasa si alguien comete un error?**  
Se puede regresar a una versión anterior fácilmente.

**¿Se necesita internet para usar Git?**  
No, funciona de forma local, aunque se puede usar internet para compartir cambios.

**¿Por qué no seguir usando archivos normales?**  
Porque Git evita errores, mantiene un historial claro y mejora el trabajo en equipo.

**¿Todos deben usar Git?**  
Sí, para que funcione correctamente, todo el equipo debe utilizarlo.
